# Sample Rejection Rubric

Use this reference when evaluating the first 1-2 generated PNG samples. Do not ask the user to approve a weak sample.

## Reject Immediately

Reject and revise the brief or prompt when any condition is true:

| Rejection label | Visible symptom | Likely fix |
|---|---|---|
| multiple conclusion zones | Title, side metric block, central ribbon, and bottom banner all read like conclusions | Collapse to one action title plus one optional takeaway |
| distracting detached metric | Large numbers sit outside the proof object and steal attention | Move numbers into the proof object or shrink to evidence chips |
| weak proof object | The page is mostly labels, cards, or boxes; the evidence carrier is not visible first | Replace with a route map, ladder, funnel, matrix, stack, bridge, or gate |
| bottom takeaway competing | Bottom band is as heavy as the title or central proof object | Lighten, shorten, or remove the takeaway |
| source note competing | Caveat/source note looks like a second takeaway | Make it small, peripheral, and quiet |
| generic dashboard | Page looks like KPI cards or SaaS dashboard | Rebuild around one proof object |
| editable-PPT screenshot feel | Thin boxes, ordinary tables, and generic icons dominate | Ask for a generated consulting exhibit, not a layout rendering |
| title scale drift | A generated page's action title looks materially larger, smaller, or differently aligned than the approved sample or sibling pages | Regenerate with the deck style contract repeated verbatim |
| inconsistent page marker | Page numbers appear on only some slides, or use different badges, positions, or sizes | Choose one page-marker system for the whole deck, then regenerate affected slides |
| visual motif drift | One slide invents a new header badge, icon style, bottom banner, source-note treatment, or material style | Restate the deck style contract and forbid new motifs |
| bottom-synthesis drift | The deck style contract calls for bottom synthesis, but some slides omit it or use a different treatment | Regenerate affected slides with the bottom-synthesis policy repeated verbatim |
| overcorrected no-takeaway | A standalone report deck omits bottom synthesis everywhere after one sample had duplicate bottom conclusions | Choose a deck-level policy; use light so-what lines where the reader needs implication |
| duplicate bottom conclusion | Evidence callout, source note, and takeaway all read like final conclusions | Keep one synthesis line, demote source note, and remove redundant callout |
| sparse concept poster | In standalone report mode, the page is clean but too empty to explain the argument without a presenter | Preserve the evidence structure and add interpretation labels, evidence bands, business implications, or source boundary around the proof object |
| one-note color | Everything is green/teal/blue with no hierarchy | Use color for orientation and status only |
| too many claims | Many caveats, conclusions, and evidence points fight for attention | Split, omit, or demote secondary material |
| text illegible | Body labels are tiny or low contrast | Reduce text and prioritize title, proof labels, and key numbers |
| evidence misattribution | A company fact, metric, or example is visually connected to a claim it does not prove | Separate observed evidence from pending validation; revise the claim-evidence map |
| false precision chart | A 2x2 or axis map compares unlike player roles on dimensions that do not fairly apply to all of them | Replace with a control-point map, layered ecosystem, or value-chain ownership map |
| label-only player evidence | Company examples appear as names plus one-word tags, so a management reader cannot tell what they show | Rewrite player labels as short evidence sentences with the implied market meaning |
| wrong visual protagonist | The most visually dominant object proves only a secondary idea, while the real message is elsewhere | Resize or change the proof object so visual weight matches the page message |
| linework overload | Boxes, borders, connectors, and pale fills give every module equal importance | Remove frames, use whitespace and light dividers, and reserve fill color for the mechanism or conclusion |

## Good Sample Signals

Acceptable samples usually have:

- Action title readable at preview size.
- One visible proof object before supporting metrics.
- Evidence anchors placed on or near the proof object.
- Company/product examples include enough text to explain why they matter, unless the audience already knows the shorthand.
- Visual connections do not imply unsupported proof; observed signals and pending validation are separated when needed.
- One clear reading path.
- Density matches the delivery mode: standalone report pages have enough evidence structure to be understood without narration.
- At most one bottom takeaway.
- Source note, if present, visibly quieter than the takeaway.
- Bottom synthesis follows the deck-level policy: absent everywhere, present everywhere, or present only on named judgment slides.
- Theme color used as orientation, not as the whole page.
- No generic card grid or ordinary table unless the table is the proof object.
- For multi-page decks, the approved sample establishes a consistent header system: title scale, title placement, subtitle relationship, page marker treatment, source-note treatment, and material style.

## Diagnostic Order

When a sample feels close but wrong, diagnose in this order:

1. Proof object: is it visible before supporting metrics?
2. Evidence attribution: does each example prove what the visual implies it proves?
3. Player readability: can a reader understand what each company/product example shows without prior context?
4. Message hierarchy: are title, subtitle, headers, numbers, and takeaway doing different jobs?
5. Competing modules: are there multiple conclusion zones?
6. Metric placement: did a true number become the headline by accident?
7. Bottom area: are takeaway and source note fighting?
8. Density fit: is a standalone report page too sparse to explain the argument?
9. Color and linework: is theme color anchoring hierarchy, or are borders/fills making every module feel equally important?
10. Deck consistency: do title scale, page marker, header motif, bottom synthesis, source note, and material treatment match the approved deck style contract?
11. Bottom synthesis: is the so-what layer useful, non-repetitive, and not competing with source notes?
12. Text fidelity: are required words and numbers legible enough?

## Regeneration Rule

Regenerate around a changed brief, not only cosmetic tweaks.

Bad fix:

```text
Make it cleaner and more premium.
```

Good fix:

```text
Keep the six-gate proof object as the only visual protagonist.
Move 27% and 91% into the evidence band below the gates, not as large side KPI blocks.
Use one bottom takeaway only; source note must be a small quiet line.
```
